Message frame templates define how the messages look like. They only define structure, however, all the content is specified in separate blocks:

  • subject: defined when writing a new letter.
  • body: as well as subject, defined when writing a letter.
  • site_url: use this variable when you need to refer to the registration system site.
  • conf_email: email for contacting organizing committee.
  • conf_site_url: URL of the conference informational site.
  • conf_full_name: full name of the conference (i.e. International Conference on Animals and Birds 2019).
  • conf_short_name: short name of the conference (i.e. ICAB'2019).
  • conf_logo_url: URL of the conference logotype. Note, that it better should be located at some well-known source (like Google Drive or Amazon S3), since some email clients may cut images from suspicious sources.

We define HTML and plain-text templates separately, because usually it is better to provide both email messages versions. No matter what type of email client our user prefers, he or she should see a nice letter! And HTML to text converters sometimes provide awful results.
